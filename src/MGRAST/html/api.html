MG-RAST Overview

RESTful Metagenomics RAST object and resource API For usage note that required parameters need to be passed as path parameters, optional parameters need to be query parameters. If an optional parameter has a list of option values, the first displayed will be used as default.

abundanceprofile

Description

A profile in biom format that contains abundance counts

info - request

Description

Returns description of parameters and attributes.

This is a synchronous GET request. This request has no parameters.

instance - request

Description

Returns a single data object.

This is a synchronous GET request.

Parameters

  • source (controlled vocabulary)

    • This parameter value can be chosen from the following (the first being default):

    • M5NR - comprehensive protein database, type organism only
    • SwissProt - protein database, type organism and feature only
    • GenBank - protein database, type organism and feature only
    • IMG - protein database, type organism and feature only
    • SEED - protein database, type organism and feature only
    • TrEMBL - protein database, type organism and feature only
    • RefSeq - protein database, type organism and feature only
    • PATRIC - protein database, type organism and feature only
    • KEGG - protein database, type organism and feature only
    • M5RNA - comprehensive RNA database, type organism only
    • RDP - RNA database, type organism and feature only
    • Greengenes - RNA database, type organism and feature only
    • LSU - RNA database, type organism and feature only
    • SSU - RNA database, type organism and feature only
    • Subsystems - ontology database, type function only
    • NOG - ontology database, type function only
    • COG - ontology database, type function only
    • KO - ontology database, type function only

    This is an optional parameter and may be passed in the query string.

  • length (int)

    • value for minimum alignment length cutoff: default is 15

    This is an optional parameter and may be passed in the query string.

  • evalue (int)

    • exponent value for maximum e-value cutoff: default is 5

    This is an optional parameter and may be passed in the query string.

  • identity (int)

    • percent value for minimum % identity cutoff: default is 60

    This is an optional parameter and may be passed in the query string.

  • type (controlled vocabulary)

    • This parameter value can be chosen from the following (the first being default):

    • organism - return organism data
    • function - return functional data
    • feature - return feature data

    This is an optional parameter and may be passed in the query string.

  • id (string)

    • unique object identifier

    This is a required parameter and must be passed as a REST parameter.

Return Attributes

  • generated_by (string)

    • identifier of the data generator

  • date (date)

    • time the output data was generated

  • matrix_type (string)

    • type of the data encoding matrix (dense or sparse)

  • data (list of lists of floats)

    • the matrix values

  • rows (list of objects)

    • This attribute has an object structure:

    • id (string) - unique identifier
    • metadata (hash) - key value pairs describing metadata

  • matrix_element_type (string)

    • data type of the elements in the return matrix

  • format_url (string)

    • url to the format specification

  • format (string)

    • format specification name

  • columns (list of objects)

    • This attribute has an object structure:

    • id (string) - unique identifier
    • metadata (hash) - list of metadata, contains the metagenome

  • shape (list of integers)

    • list of the dimension sizes of the return matrix

  • type (string)

    • type of the data in the return table (taxon, function or gene)

  • id (string)

    • unique object identifier

analysisset

Description

An analysis file from the processing of a metagenome from a specific stage in its analysis

info - request

Description

Returns description of parameters and attributes.

This is a synchronous GET request. This request has no parameters.

instance - request

Description

Returns a single sequence file.

This is a synchronous GET request.

Parameters

  • id (string)

    • unique analysis set identifier - to get a list of all identifiers for a metagenome, use the setlist request

    This is a required parameter and must be passed as a REST parameter.

Return Attributes

  • data (file)

    • requested analysis file

setlist - request

Description

Returns a list of sets of sequence files for the given id.

This is a synchronous GET request.

Parameters

  • id (string)

    • unique metagenome identifier

    This is a required parameter and must be passed as a REST parameter.

Return Attributes

  • stage_name (string)

    • name of the stage in processing of this file

  • file_name (string)

    • name of the analysis file

  • url (string)

    • url for retrieving this analysis file

  • stage_type (string)

    • type of the analysis file within a stage, i.e. passed or removed for quality control steps

  • id (string)

    • unique identifier of the analysis file

  • stage_id (string)

    • three digit numerical identifier of the stage

library

Description

A library of metagenomic samples from some environment

info - request

Description

Returns description of parameters and attributes.

This is a synchronous GET request. This request has no parameters.

query - request

Description

Returns a set of data matching the query criteria.

This is a synchronous GET request.

Parameters

  • verbosity (controlled vocabulary)

    • This parameter value can be chosen from the following (the first being default):

    • minimal - returns only minimal information

    This is an optional parameter and may be passed in the query string.

  • order (controlled vocabulary)

    • This parameter value can be chosen from the following (the first being default):

    • id - return data objects ordered by id
    • name - return data objects ordered by name

    This is an optional parameter and may be passed in the query string.

  • limit (integer)

    • maximum number of items requested

    This is an optional parameter and may be passed in the query string.

  • offset (integer)

    • zero based index of the first data object to be returned

    This is an optional parameter and may be passed in the query string.

Return Attributes

  • next (uri)

    • link to the previous set or null if this is the first set

  • prev (uri)

    • link to the next set or null if this is the last set

  • order (string)

    • name of the attribute the returned data is ordered by

  • data (list of objects)

    • This attribute has an object structure:

    • project (reference project) - reference to the project object
    • version (integer) - version of the object
    • name (string) - human readable identifier
    • sequencesets (list of reference sequencesets) - a list of references to the related sequence sets
    • metagenome (reference metagenome) - reference to the related metagenome object
    • created (date) - time the object was first created
    • url (uri) - resource location of this object instance
    • metadata (hash) - key value pairs describing metadata
    • sample (reference sample) - reference to the related sample object
    • id (string) - unique object identifier

  • limit (integer)

    • maximum number of data items returned, default is 10

  • total_count (integer)

    • total number of available data items

  • offset (integer)

    • zero based index of the first returned data item

instance - request

Description

This is a synchronous GET request.

Parameters

  • verbosity (controlled vocabulary)

    • This parameter value can be chosen from the following (the first being default):

    • minimal - returns only minimal information
    • verbose - returns a standard subselection of metadata
    • full - returns all connected metadata

    This is an optional parameter and may be passed in the query string.

  • id (string)

    This is a required parameter and must be passed as a REST parameter.

Return Attributes

  • project (reference project)

    • reference to the project object

  • version (integer)

    • version of the object

  • name (string)

    • human readable identifier

  • sequencesets (list of reference sequencesets)

    • a list of references to the related sequence sets

  • metagenome (reference metagenome)

    • reference to the related metagenome object

  • created (date)

    • time the object was first created

  • url (uri)

    • resource location of this object instance

  • metadata (hash)

    • key value pairs describing metadata

  • sample (reference sample)

    • reference to the related sample object

  • id (string)

    • unique object identifier

m5nr

Description

M5NR provides data through a comprehensive non-redundant protein / rRNA database

info - request

Description

Returns description of parameters and attributes.

This is a synchronous GET request. This request has no parameters.

ontology - request

Description

This is a synchronous GET request.

Parameters

  • source (controlled vocabulary)

    • This parameter value can be chosen from the following (the first being default):

    • Subsystems - returns 4 level SEED-Subsystems ontology
    • COG - returns 3 level COG ontology
    • NOG - returns 3 level NOG ontology
    • KO - returns 4 level KEGG-KO ontology

    This is an optional parameter and may be passed in the query string.

  • parent_name (string)

    • name of ontology group to retrieve children of

    This is an optional parameter and may be passed in the query string.

  • min_level (controlled vocabulary)

    • This parameter value can be chosen from the following (the first being default):

    • function - bottom ontology level
    • level3 - ontology level
    • level2 - ontology level
    • level1 - top ontology level

    This is an optional parameter and may be passed in the query string.

Return Attributes

This request returns a list, where each element is a list. The elements of this list contain requested ontology levels, from highest to lowest.

taxonomy - request

Description

This is a synchronous GET request.

Parameters

  • parent_name (string)

    • name of taxanomy group to retrieve children of

    This is an optional parameter and may be passed in the query string.

  • min_level (controlled vocabulary)

    • This parameter value can be chosen from the following (the first being default):

    • species - taxonomy level
    • genus - taxonomy level
    • family - taxonomy level
    • order - taxonomy level
    • class - taxonomy level
    • phylum - taxonomy level
    • domain - top taxonomy level

    This is an optional parameter and may be passed in the query string.

Return Attributes

This request returns a list, where each element is a list. The elements of this list contain requested taxonomy levels, from highest to lowest.

sources - request

Description

This is a synchronous GET request.

Parameters

    Return Attributes

    This request returns a list, where each element is a hash. They key of the hash is the source name, the value is a object.
